# 09 · Settings / админка

> Все экраны настроек: профиль, команда, источники данных, sticky-правила, шаблоны, биллинг, интеграции, аудит.

## Когда применять

- «настройки», «settings», «админка»
- «управление пользователями», «команда / роли»
- «интеграции», «webhooks», «API»
- «биллинг», «подписка», «лимиты»
- «аудит», «лог действий»

## Принцип

Настройки в B2B-инструменте — **самый большой раздел**. Не пытайся ужать всё в одну страницу с табами. Используй паттерн **«вертикальный sub-rail слева, контент справа»**, на манер GitHub Settings или Linear.

## Sub-rail (240px)

Тот же стилевой код, что у context list в [01-list.md](01-list.md), но с группами по существительным верхнего уровня.

### Группы (закрытый набор)

```
ОРГАНИЗАЦИЯ
  ├ Профиль организации
  ├ Реквизиты и бланки
  └ Биллинг и подписка

КОМАНДА
  ├ Пользователи (12)
  ├ Роли и права
  └ Приглашения (3)

ДАННЫЕ
  ├ Источники (почта, формы, телефония)
  ├ Шаблоны писем
  ├ Sticky-правила
  ├ Каталог брендов
  └ Артикулы и аналоги

ИНТЕГРАЦИИ
  ├ API ключи
  ├ Webhooks
  └ Экспорт / импорт CSV

ЖУРНАЛ
  ├ Аудит действий
  └ События системы

МОЙ ПРОФИЛЬ
  ├ Личные данные
  ├ Уведомления
  └ Безопасность (2FA, сессии)
```

- Группы — `group-label` 10.5px uppercase fg-3, padding 14px 20px 6px.
- Пункт 28px, padding 6px 20px, hover `--bg-hover`, active inset 2px accent left.
- Счётчик справа — mono 11.5px fg-3. Если требует внимания (3 приглашения ждут принятия) — pill `--amber-50` / `--amber-700`.
- **Точка `●`** перед пунктом, который содержит **несохранённые изменения** на этой сессии — `--amber-500`. Чтобы пользователь видел недосейв при переключении.

## Main pane

### Header страницы

```
Реквизиты и бланки
последний раз сохранено: пт 22 ноя · 14:32 · Иванов А.

                                                       [Экспорт PDF] [···]
```

- H1 22px semibold.
- Meta — fg-3 11.5px, `последний раз сохранено: <date> · <user>` mono для даты и user-инициалов.
- Справа — secondary actions если применимо (export, импорт).

### Section cards

Каждая section — отдельная card 1px border `--border`, padding 20px 24px. Между cards 16px gap.

Шапка card 36px:

```
h2 14px semibold     ●          meta fg-3 11.5px         [···]
                  unsaved
```

- Точка перед h2 — `--amber-500` 8px, появляется когда в card есть dirty fields. Title — `несохранённые изменения`.
- Meta — `последний апдейт: <relative time>` или `активно 8 правил`.

### Save UX (важно!)

В settings — **autosave-by-section**, не глобальный `Сохранить`.

- На blur поля или после 800ms idle — отправляем PATCH.
- В шапке card во время сохранения — chip `сохраняется…` (sky-50, fg-2 11.5px medium).
- После — chip `сохранено` (emerald-50, fg-2) на 2 секунды, затем растворяется.
- Если упало — chip `не сохранено` (red-50, accent text) с `[повторить]` button. Card получает `border-color: var(--amber-500)`.

**Исключения** — поля где autosave опасен (reset password, удалить аккаунт): отдельная form со своим `[Применить]` button.

## Типичные паттерны секций

### A · Form-секция (профиль, реквизиты)

См. [04-forms-modals.md](04-forms-modals.md) для row-стиля. Внутри settings:
- Все поля — 36px high.
- Двухколоночный layout 50/50 если поля парные (ИНН + КПП, Город + Индекс).
- Helper-text под полем 11.5px fg-3 — указывает source (`из ЕГРЮЛ, обновлено 19.10`) или формат (`12 цифр, ИП — 10 или 12`).

### B · Table-секция (пользователи, ключи, аудит)

Такой же pattern, что [01-list.md](01-list.md), но:
- Без context-rail (он же sub-rail слева).
- Filter-bar спрятан, есть только search-input + 1–2 pre-filter chip.
- Footer pagination проще (только prev/next).
- Row-actions `···` обязательно.

### C · Toggle-list секция (уведомления, права роли)

```
┌────────────────────────────────────────────────┐
│ Уведомления о SLA                              │
│ Email если заявка не отвечена 24+ ч.    [ ⬤  ] │ ← toggle
├────────────────────────────────────────────────┤
│ Уведомления о КП                               │
│ Email когда поставщик прислал КП.       [⬤   ] │
├────────────────────────────────────────────────┤
│ Дайджест в 9:00                                │
│ Раз в день, по будням.                  [⬤   ] │
└────────────────────────────────────────────────┘
```

- Каждый row 56px, 16px y-padding.
- Левая часть: title 13px medium + helper 12px fg-3.
- Toggle справа, vertical-center.
- Hairline между rows.

### D · Two-pane editor (sticky-правила, шаблоны)

Левая колонка — список (240px), правая — редактор выбранного. См. готовый экран `ui_kits/crm/05-mail-rules.html`. Подходит когда правила/шаблоны 5+ штук.

## Permissions / роли

Если экран зависит от роли:
- Запрещённые секции в sub-rail — не скрывай, **показывай задизейбленными** с иконкой замка 12px и tooltip `Доступно роли: владелец, админ`.
- Запрещённое действие в форме — поле readonly, hint под ним: `Изменить может: владелец`.
- 403 при попытке submit — баннер `Нет прав на это действие. [Запросить у админа]` (см. [05-empty-loading-error.md](05-empty-loading-error.md)).

## Danger zone (внизу страницы)

Карточка с разделителем-надписью `DANGER ZONE` (uppercase 11.5px medium fg-3, 24px y-margin сверху):

```
─── DANGER ZONE ─────────────────────────────────────

┌─ red border 1px ──────────────────────────────────┐
│ Удалить организацию                               │
│ Все данные удалятся безвозвратно. Восстановить    │
│ нельзя.                              [Удалить]    │
├───────────────────────────────────────────────────┤
│ Сменить владельца                                 │
│ Передать root-доступ другому пользователю.        │
│                              [Сменить владельца]  │
└───────────────────────────────────────────────────┘
```

- Card border 1px `--red-300`.
- Body 16px y, разделители hairline `--red-100`.
- Action button — `border-color: var(--accent); color: var(--accent); background: var(--bg-surface);` (red ghost).
- Confirm обязательно с **типом текста-подтверждением**: `Введите название организации, чтобы подтвердить удаление.` — input должен совпасть с текущим именем.

## Copywriting

- **H1 страницы** — существительное-объект, не глагол: `Реквизиты и бланки`, не `Изменить реквизиты`.
- **H2 секции** — то же: `Реквизиты юр.лица`, `Подпись и печать`, `Логотип в шапке`.
- **Helper под полем** — фактический формат: `12 цифр, ИП — 10 или 12 цифр.`
- **Успешный save** — `сохранено` (lowercase, без `!`).
- **Failed save** — `не сохранено · [повторить]`.
- **Permission helper** — `Изменить может: владелец, админ`.
- **Danger zone confirm** — `Введите «MyLift CRM», чтобы подтвердить.` (точное название в кавычках).

## Do / Don't

| ✅ Do | ❌ Don't |
|---|---|
| Sub-rail слева 240px с группами | Все настройки одной страницей с якорями |
| Autosave-by-section, chip `сохранено` | Глобальный `Сохранить` снизу страницы |
| Permissions readonly + lock-icon | Скрыть запрещённые пункты совсем |
| Danger zone — отдельный red-border блок | Кнопка `Удалить` среди обычных action |
| Confirm удаления — введи имя | Confirm — `Вы уверены?` чек-бокс |
| Уведомления — toggle list | Уведомления — table с edit-row |
| Точка `●` амбер на dirty pages в rail | Глобальный banner «Сохраните изменения» |

## Edge cases

- **Multiple sessions** — пользователь редактирует на двух устройствах: на blur мы получаем 409 conflict; chip `обновлено в другой вкладке · [принять / переписать]`.
- **Зависимая настройка** — если выключен «Email-источник», секция «Sticky-правила почты» помечается `неактивно (нет email-источника). [Подключить]`.
- **Demo data** — в settings есть отдельная секция: `Демо-данные · 12 заявок-примеров · [Удалить все demo]`. Кнопка red ghost.
- **Несохранённый dirty** при попытке закрыть вкладку — `beforeunload` confirm. Точно знаем какие именно секции — пишем в confirm: `Несохранённые изменения в: Реквизиты, Уведомления.`
- **Audit log** — это read-only table с фильтром по дате/пользователю/действию. Действие сериализовано: `<actor> <verb> <object>` (`Иванов А. изменил реквизиты организации`). Mono для timestamp, fg-1 для actor, fg-2 для verb+object.
- **API key** — показывается **один раз** при создании, потом маскируется (`mzc_••••••8f3a`). Кнопка `Скопировать` рядом во время первого показа.
- **Webhook secret** — то же. Плюс — кнопка `Сгенерировать новый` (revokes old + confirm).
