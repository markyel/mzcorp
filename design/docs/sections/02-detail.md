# 02 · Detail view (карточка)

> Карточка заявки, клиента, поставщика, письма, артикула, КП.

## Когда применять

- «карточка заявки», «детали заявки MZ-21384»
- «открой клиента», «страница поставщика»
- «просмотр письма», «карточка артикула»

Один объект во всю ширину контента. Если объектов два рядом (split master/detail) — это **inspector pattern**, см. блок Inspector в [01-list.md](01-list.md), а не этот документ.

```

│    │  meta-row: клиент · менеджер · сумма · SLA · теги             │
│    │  primary actions row (3–5 кнопок)                             │

│ I  │ ┌─ section card ─────┐    │ ┌─ Stage / Workflow ────────────┐ │
```

## Слои сверху вниз

### 1 · Sticky header (80–96px)
Это самый важный блок страницы. Должен дать 5 фактов за полсекунды:

1. **Что это** (тип + код): `Заявка · MZ-21384` — тип fg-3 12.5px, код mono 14px fg-1.
2. **Тема одной строкой** — h1 20px semibold. Длинная — обрезай ellipsis, full в tooltip.
3. **Статус** — chip справа от h1 (lowercase, из закрытого набора).
4. **Возраст / SLA** — `4 дн · до SLA 1 дн` mono fg-3, красный если over.
5. **Меню `···`** — переход в Inbox, дублировать, экспортировать, закрыть.

Под h1 идёт **meta-row** (12.5px fg-3, разделители `·`):
```
ТД «Лифт-Сервис» · менеджер: Иванов А. · 142 800 ₽ · KONE · 4 позиции · #refresh #sticky
```

Под meta — **primary actions** (3–5 кнопок). Слева направо: главная, рядом 1–2 связанные, справа deconstructive (если есть).
- 1 primary (red): «Отправить КП» / «Сформировать счёт»
- 2 secondary: «Запросить refresh цен», «Назначить менеджера»
- 1 ghost справа: «Поставить на паузу»

Sticky при скролле основной колонки. Тени нет; держится на `border-bottom: 1px solid var(--border)`.

### 2 · Tabs (36px)
- Inline-flex, gap 4px, padding 0 20px.
- Active — `border-bottom: 2px solid var(--accent)`, fg-1 semibold. Inactive — fg-3 medium.
- Counts in tabs — `Переписка 12` mono 11.5px fg-3.
- Не больше 6 видимых таб; остальное — `··· ⌄` overflow.

**Don't:** табы не должны прыгать (фиксируй высоту 36px). Не делай pill-табы — они конфликтуют с chip-ами.

### 3 · Main column · section card

Каждая секция — карточка `1px solid var(--border)`, `border-radius: var(--r-md)`, `background: var(--bg-surface)`. Между карточками 16px gap.

Шапка карточки 44px:
```
[h2 14px semibold] [chip · опц.]              [meta fg-3 11.5px] [···]
```

Содержимое — таблица / форма / KV-список / inline timeline. Padding контента 16px (или 0 если внутри таблица — таблица идёт от края до края карточки).

#### Типичные секции

- **Обзор** — два столбца KV (key fg-3 12px, value fg-1 13px). Колонки 50/50.
- **Позиции** — таблица позиций в каталоге. См. шаблон таблицы из 01-list (компактный density).
- **Переписка** — список писем (28px row mono subject + sender + дата) → клик раскрывает inline.
- **Поставщики** — параллельная таблица: 1 строка = 1 поставщик с КП, колонки: поставщик · цена · срок · разница к лучшему · чат · действие.
- **История** — vertical timeline, см. компонент в `colors_and_type.css` блоке timeline.
- **Файлы** — grid 3-кол attachments (превью 64×64 + filename + size + meta).

### 4 · Aside (320px)
Не дублирует main, **компрессирует**. Три блока:

#### a) Workflow / Stage stepper
Vertical stepper с 5–7 шагами пайплайна: `получено → квалификация → запрос цен → КП → счёт → оплата → отгрузка`. Текущий шаг — фон `--sky-50`, точка `--sky-500`. Пройденные — точка `--emerald-500`. Будущие — точка fg-3 outline.

#### b) Properties (KV-list)
Все техно-факты, которые **не помещаются** в meta-row хедера:
- Источник (форма/звонок/email/whatsapp)
- ИНН клиента (mono)
- Канал ответа
- Привязанный проект
- Теги
- Создано / обновлено (mono timestamp)

Row 28px, label 100px fg-3 11.5px uppercase, value fg-1 12.5px. Ховер row не нужен.

#### c) Activity / inline log
Лента событий (последние 6, see-more раскрывает). Каждое событие — 2 строки: `actor → action target` + mono timestamp fg-3. Без иллюстраций аватарок, только инициалы 22px.

## Плотность

| Variant | Когда | Aside | Section padding |
|---|---|---|---|
| **comfortable** | первая карточка после регистрации, мобила | 360px | 20px |
| **default** | прод | 320px | 16px |
| **compact** | поддержка / B2B-оператор | 280px (или off-canvas) | 12px |

В compact aside становится drawer'ом (кнопка в шапке `Свойства`) — main колонка занимает всю ширину.

## Copywriting

- **H1** — фактическая тема заявки, как пишет клиент: `Замок ДШ Metron 2PTL левый, 4 шт, KONE`. Не редактируй формулировки автогенератором.
- **Action labels** — глагол + существительное: `Отправить КП`, `Запросить refresh цен`, `Поставить на паузу`. Не «КП», не «Refresh».
- **Status в chip** — lowercase, из закрытого набора (см. корневой README). `на согласовании`, `КП отправлено`, `refresh цен`.
- **Empty section**: 1 строка fg-3 + inline-CTA: `Позиций ещё нет. [Добавить из каталога]`.
- **Confirm на destructive**: всегда с темой объекта в кавычках: `Закрыть заявку «Замок ДШ Metron 2PTL»?`

## Do / Don't

| ✅ Do | ❌ Don't |
|---|---|
| Sticky header с кодом и статусом | Хедер скроллится вместе с контентом |
| 1 primary CTA (красный) | 2+ красных кнопки в action row |
| Aside — properties + activity + workflow | Aside — рекламные блоки / hint-карточки |
| Tabs с counts mono | Tabs с цветными бейджами в углу |
| Section card 16–20px padding | Card 24+ padding (это data UI) |
| Workflow — stepper из закрытого набора | Свободный набор статусов в stepper |

## Edge cases

- **Нет клиента (orphan)**: на месте имени клиента в meta — `— не привязан` fg-3 italic, clickable → quick-attach modal.
- **Несколько связанных писем** — в табе `Переписка` верхняя строка `Тред · 12 писем · 4 участника`.
- **КП ещё нет** — таб «КП» отсутствует, в Workflow шаг `КП` подсвечен серым с подсказкой `Сформируйте, чтобы перейти дальше`.
- **Сотни позиций** — таблица позиций показывает 20, ниже `Показать ещё 487`. Не пагинируй внутри карточки.
- **Просрочено** — chip `просрочено` + красный возраст в шапке + красная вертикальная полоска `inset 3px 0 0 var(--accent)` слева от sticky-хедера.
- **Заархивировано / закрыто** — карточка читается, но action row блокируется единственной кнопкой `Открыть заново`.

- [ ] Sticky header содержит код (mono), h1, chip-status, возраст

- [ ] Destructive action — последняя в action row, ghost-style до confirm
