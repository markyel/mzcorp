# 07 · Onboarding (первый запуск)

> Всё, что показывается пользователю в первые минуты работы с MyLift CRM. Не туториал, а конкретные шаги до первой ценности.

## Когда применять

- «онбординг», «первый запуск», «get started»
- «помоги пользователю настроить»
- «приветственный экран»

В CRM-инструменте онбординг **НЕ** объясняет, что такое CRM. Объясняет три вещи:

1. **Кто ты** в системе (роль, команда)
2. **Откуда придут заявки** (почта/форма/звонок)
3. **Где первая заявка** (создай тестовую или жди реальную)

Всё остальное — мешает. Никаких туров по 12 шагов с пузырьками. Никаких видео.

## Три фазы

| Фаза | Когда | Длительность |
|---|---|---|
| **A · Setup wizard** | Самый первый вход | 3–4 шага · ~3 мин |
| **B · Empty workspace** | Setup закончен, ещё нет данных | пассивно, до первой заявки |
| **C · Inline hints** | Первые 5 действий в продакшен-UI | по событию, не proactive |

## A · Setup wizard

### Anatomy

Полноэкранный layout, по центру — карточка 560px width, тонкий progress сверху.

```
┌──────────────────────────────────────────────────────┐
│ MyLift CRM · setup                                    │ ← topbar 48px, без rail
├──────────────────────────────────────────────────────┤
│  ●───●───○───○      Шаг 2 из 4                       │ ← progress 4 точки
│                                                      │
│           ┌──────────────────────────────┐           │
│           │ h1 · 22px semibold           │           │
│           │ помощь в одну строку         │           │
│           │                              │           │
│           │ form rows…                   │           │
│           │                              │           │
│           │  [Пропустить]    [Далее →]   │           │
│           └──────────────────────────────┘           │
│                                                      │
│  fg-3 11.5px · «Это можно изменить позже в …»       │
└──────────────────────────────────────────────────────┘
```

### 4 шага (закрытый набор)

#### Шаг 1 · Кто ты
- H1: `Расскажите о себе.`
- Поля: ФИО · должность (select: руководитель · менеджер · логист · бухгалтер · админ) · телефон.
- Email уже подставлен из логина, mono fg-3 readonly.
- Skip нет — это identity.

#### Шаг 2 · Команда / организация
- H1: `Где вы работаете?`
- Поля: организация (search → existing or create) · ИНН (если новая) · кол-во менеджеров (3 / 5–10 / 10+).
- На основе ответов CRM сама создаёт нужное число пустых учёток-приглашений.
- Skip = «работаю один» (CRM прячет teams-фичи).

#### Шаг 3 · Источники заявок
- H1: `Откуда приходят заявки?`
- Чеклист (multi):
  - ☐ почта `*@mylift.ru` (подключить → OAuth Yandex/Mail)
  - ☐ форма на сайте mylift.ru (показать сниппет JS, скопировать)
  - ☐ телефон (подключить → телефония-провайдер · Mango / UIS / Beeline)
  - ☐ WhatsApp / Telegram (показать `@mylift_bot`, инструкция)
- Под каждым — fg-3 11.5px помощь: `Подключим IMAP, импортируем последние 30 дней.`
- Skip = «настрою позже», даёт пустой workspace.

#### Шаг 4 · Готово
- H1: `Готово. Создаём 12 заявок-примеров?`
- Toggle: `создать demo-данные` (default: on).
  - Если on → 12 синтетических заявок MZ-90001..MZ-90012, легко удалить через `Settings → Demo data → Delete all demo`.
- Кнопка `[Открыть CRM]` primary.

### Правила wizard

- **Никаких «подсказок»-bubble** на этом этапе. Всё в формах.
- **Прогресс — 4 точки**, не bar (см. multi-step в [04-forms-modals.md](04-forms-modals.md)).
- **Skip / Назад** — ghost-buttons. Skip ведёт на следующий шаг, Назад — на предыдущий с сохранением.
- Progress + back persisted в localStorage. Перезагрузка вернёт на тот же шаг.
- **Тон** — деловой, без хайпа. Не `Добро пожаловать! 🎉`, не `Давайте начнём!`. Просто `Расскажите о себе.`

## B · Empty workspace

После wizard. Пользователь видит обычный CRM-layout, но списки пустые. Здесь работает [05-empty-loading-error.md](05-empty-loading-error.md), но с **дополнительным «next-step»-блоком**.

### Layout

```
┌──────────────────────────────────────────────────────┐
│ TOPBAR · RAIL (как обычно)                           │
├────┬─────────────────────────────────────────────────┤
│ R  │ ┌─ NEXT STEPS ─────────────────────────────┐    │
│    │ │ Чеклист 4 пунктов · 1px border           │    │
│ A  │ │   ☐ подключите почту           [сделать] │    │
│    │ │   ☐ пригласите менеджеров     [сделать] │    │
│ I  │ │   ☑ настроены источники                  │    │
│    │ │   ☑ создан demo-набор                    │    │
│ L  │ └──────────────────────────────────────────┘    │
│    │                                                  │
│    │ ┌─ список заявок (с demo-данными) ─────────┐    │
│    │ └──────────────────────────────────────────┘    │
└────┴─────────────────────────────────────────────────┘
```

### Правила NEXT STEPS блока

- Card 1px border, padding 16px 20px, **не выделяется цветом** (не sky-50, не amber-50). Это просто карточка.
- H2 в шапке: `Следующие шаги · 2 из 4`. Прогресс — текстом, не bar.
- Каждая строка 36px:
  - checkbox 14×14 (выкл — `--border-strong`, вкл — `--accent` fill check),
  - текст 13px medium fg-1,
  - справа кнопка `[сделать]` ghost 24px высоты для выкл, mono `пт 22 ноя` fg-3 для сделанных.
- Card имеет `[×]` в правом верхнем углу — спрятать навсегда. Не показывает confirm.
- Скрыть **автоматически**, когда все 4 = ☑.

### Не делай

- ❌ NEXT STEPS поверх контента как pop-over
- ❌ Procentage bar `60% настроено`
- ❌ Геймификация (badge, очки, звёзды)
- ❌ «Отлично! Вы прошли 50%! 🎯»

## C · Inline hints (первые 5 действий)

Показываются **по событию**, не proactive. Каждый — одна стрелка-tooltip, появляется при наведении / фокусе / первом hover в нужной зоне.

### Когда показываем

| Событие | Подсказка |
|---|---|
| Первый раз навёл на чип-фильтр | `Чип = конкретное значение фильтра. Кликните, чтобы добавить.` |
| Первый раз выделил строку | `Выделите ещё, чтобы появились массовые действия сверху.` |
| Первый раз открыл карточку заявки | `Слева — пайплайн. Сейчас здесь работа: переписка, цены, КП.` |
| Первый раз попал в Settings | `Большинство правил живут в Mail rules.` |
| Первый раз увидел красный chip-status | `Красный — просрочено по SLA. Этот клиент ждёт > 48 ч.` |

### Tooltip правила

- Размер max-width 280px, padding 10px 12px.
- Фон `--fg-1`, текст `#fff`, mono цвет — нет (это разъяснение, не код).
- Стрелка 6×6, цвет тот же.
- 13px regular line-height 1.4.
- 1 CTA внутри — link-style underline `Понятно`. Клик — навсегда скрывает этот hint и пишет в localStorage `hint:<id>:dismissed`.
- Auto-dismiss через 12 секунд если не interacted.

### Никогда

- ❌ Tour из 8 шагов с next/prev
- ❌ Backdrop с дыркой (spotlight)
- ❌ Hint при каждом hover (только первый раз)
- ❌ Hint поверх контента, который пользователь сейчас читает

## Copywriting

- **H1 wizard** — императив-вопрос: `Расскажите о себе.`, `Где вы работаете?`. Не `Шаг 1: данные пользователя`.
- **Помощь под полем** — фактическая, не воодушевляющая: `Подключим IMAP, импортируем последние 30 дней.`, не `Это поможет вам быстрее обрабатывать почту!`.
- **CTA «Готово»** — `[Открыть CRM]`, не `Начать работу!`.
- **Демо-данные** — `12 заявок-примеров` (точное число, не «несколько»).
- **NEXT STEPS** — глагол-инфинитив: `подключите почту`, `пригласите менеджеров`. С маленькой буквы, без точки.
- **Inline hint** — 1 предложение факта, без `Совет:`, без `Знаете ли вы…`.

## Do / Don't

| ✅ Do | ❌ Don't |
|---|---|
| 4 шага wizard, фиксированно | «Гибкий» wizard от 3 до 10 шагов |
| Skip ghost-button на каждом шаге (кроме identity) | Заставить пройти все шаги |
| NEXT STEPS — обычная card в workspace | NEXT STEPS — модалка / pop-over |
| Inline hint только при первом событии | Hint при каждом hover |
| Демо-данные с явной меткой и `delete all` | Демо-данные неотличимы от реальных |
| Тон деловой, без хайпа | `🚀 Добро пожаловать!` |

## Edge cases

- **Пользователь закрыл вкладку на шаге 3** — при следующем входе wizard восстанавливается на том же шаге, заполненные поля сохранены.
- **Email-источник OAuth-fail** — inline-error на шаге 3 + `[Пропустить, настрою позже]` ghost становится primary.
- **Уже есть заявки в системе** (приглашён в существующий workspace) — wizard сокращён до **1 шага**: `Кто ты`, дальше сразу в продакшен.
- **Реinviting** — у инвайтнутого менеджера wizard покажет 2 шага: identity + `присоединиться к команде [...]` с подтверждением.
- **Демо-данные забыли удалить** — через 30 дней показать однострочный inline-баннер сверху списка `У вас всё ещё 12 demo-заявок. [Удалить все]` (амбер-баннер, см. [05-empty-loading-error.md](05-empty-loading-error.md)).
- **Onboarding для нового члена команды** — wizard короткий (только identity), NEXT STEPS свой («познакомьтесь с пайплайном», «первая назначенная заявка»).
