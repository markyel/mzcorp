# 06 · Email templates & переписка

> Шаблоны писем (для отправки клиенту/поставщику) и интерфейс переписки внутри CRM.

## Когда применять

- «шаблон письма», «email template»
- «переписка», «тред», «inbox карточки»
- «отправить КП письмом», «нотификация по почте»
- «sticky-правило» / автоответ

Шаблоны — это **исходящие письма из CRM**, рендерятся как HTML-email. Переписка — это **просмотр исходящих и входящих** внутри карточки заявки или поставщика.

## Anatomy · Email template (рендер для клиента)

```
┌─────────────────────────────────────────────┐ 600px width фикс
│ HEADER · 12px y · brand mark · slogan-line  │
│   логотип 24px · «MyLift · поставщик…»     │
├─────────────────────────────────────────────┤
│ BODY · 24px x · 20px y                      │
│   приветствие 1 строкой                     │
│   1–2 параграфа сути (13px serif или sans)  │
│   ┌─ DATA TABLE (если КП/счёт) ─────────┐   │
│   │ позиции · цена · срок                │   │
│   └────────────────────────────────────┘   │
│   CTA inline-button (одна, primary)         │
│   подпись · должность · контакты            │
├─────────────────────────────────────────────┤
│ FOOTER · 12px y · fg-3 11px                  │
│   юр. данные · ИНН · ссылка отписки         │
└─────────────────────────────────────────────┘
```

### Правила email HTML

- **Ширина** — 600px (стандарт mail-клиентов), `<table role="presentation">` за основу, не flex/grid (Outlook).
- **Inline styles only** — никаких `<style>` блоков и `var(--*)`. Все цвета — hex, все размеры — px.
- **Шрифт** — `font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', Arial, sans-serif;`. Не Inter (не везде есть).
- **Кнопка CTA** — `<a>` с padding 12px 20px, `background:#C1152E; color:#fff; text-decoration:none; border-radius:6px;`. Не `<button>`.
- **Цвета** — упрощённый набор:
  - текст: `#0F1419` body, `#5C6975` muted
  - линии: `#E5E8EB`
  - accent: `#C1152E` (только CTA + статус-маркер)
  - bg: `#FFFFFF` body, `#F4F6F8` data table header
- **Логотип** — отдельный hosted PNG/JPG, не SVG (Outlook не любит).
- **Тёмная тема mail** — не пытайся поддерживать; используй only-light палитру и явный `background-color` на каждом блоке.

## Шаблоны (выучить структуру)

### A · КП клиенту

```
[logo] MyLift

Здравствуйте, Алексей.

Подготовили КП по вашему запросу от 22.11. По всем 4 позициям
есть наличие, отгрузка из Москвы.

┌─────────────────────────────────────────────────┐
│ ПОЗИЦИЯ            КОЛ-ВО   ЦЕНА      СУММА     │
├─────────────────────────────────────────────────┤
│ КВШ D320 KONE      4 шт     34 200 ₽  136 800   │
│ Плата LCEFCB       1 шт      6 000 ₽    6 000   │
│ …                                               │
├─────────────────────────────────────────────────┤
│ ИТОГО без НДС                          142 800  │
│ ИТОГО с НДС 20%                        171 360  │
└─────────────────────────────────────────────────┘

КП действительно до 28.11. Срок отгрузки — 3 рабочих дня.

[ Согласовать и оплатить ]   ← кнопка accent

—
Иван Петров · менеджер MyLift
+7 495 …  ·  i.petrov@mylift.ru
```

- Тема: `КП MZ-21384 · 4 позиции · 142 800 ₽ · до 28.11`
- В preheader (первая скрытая строка): `Действительно до 28.11. Срок 3 рабочих дня.`
- 1 CTA, не больше. Вторичные действия — текст-ссылками внизу подписи.

### B · Запрос refresh цен поставщику

```
Здравствуйте, отдел продаж WITTUR.

Прошу подтвердить актуальность цен и сроков по позициям ниже.
Запрос #MZ-21384, клиент — ТД «Лифт-Сервис».

[таблица позиций без сумм — только артикул, кол-во, ваш прайс]

Если есть аналог при отсутствии — напишите, пожалуйста, в ответе.
Срок ответа — до пт 24.11 18:00 МСК.

—
…
```

- Тема: `Refresh цен · MZ-21384 · 4 поз. · до 24.11`
- Без CTA-кнопки, ответ ожидается в виде reply.

### C · Sticky-уведомление (автоматическое)

```
Заявка MZ-21384 ждёт вашего ответа уже 3 дня.

Статус: КП отправлено, клиент не ответил.
Пора дёрнуть напоминанием. Шаблон ниже:

  [ Открыть в CRM ]

—
автомат от MyLift CRM
```

- Тема: `MZ-21384 · 3 дня без ответа клиента`
- Тон — нейтральный, не панический. Без `!!!`.

### D · Внутренняя нотификация (назначен менеджер)

Минималистично:

```
Вам назначена заявка MZ-21384.
ТД «Лифт-Сервис» · 4 поз. · KONE · 142 800 ₽

[ Открыть в CRM ]
```

- Тема: `Назначено: MZ-21384 (ТД «Лифт-Сервис»)`

## Anatomy · Переписка внутри карточки

В карточке заявки таб **Переписка**:

```
┌──────────────────────────────────────────────────────┐
│ tab: Переписка · 12 писем · 4 участника · [новое]    │
├──────────────────────────────────────────────────────┤
│ ┌─ thread row · 56px ─────────────────────────────┐ │
│ │ avatar · sender → recipient · subject · time   │ │
│ │ snippet · attachments badge                    │ │
│ └─────────────────────────────────────────────────┘ │
│ ┌─ EXPANDED message ──────────────────────────────┐ │
│ │ headers (from/to/cc/date) · mono                │ │
│ │ body · 13px sans · sanitized HTML или plain    │ │
│ │ attachments grid                                │ │
│ │ [ответить] [переслать] [отметить как…]         │ │
│ └─────────────────────────────────────────────────┘ │
└──────────────────────────────────────────────────────┘
```

### Правила thread row

- Высота 56px, gap 12px, hover `--bg-hover`.
- Avatar 28px (инициалы).
- Sender → recipient: `Иванов А. → отдел продаж WITTUR` 12.5px medium fg-1, стрелка fg-3.
- Subject: 13px medium fg-1 после sender, обрезается ellipsis.
- Time справа mono 11.5px fg-3 (`пт 14:32`).
- Snippet — fg-3 12px, одна строка ellipsis, под subject.
- Attachments badge — `📎 3` (символ скрепки, mono 11.5px) после subject.
- **Прочитанное** — fg-2; **непрочитанное** — fg-1 + кружок `--accent` 6px слева.

### Expanded message

- Headers — KV-list 24px rows (см. detail · properties), вверху expanded.
- Body — `prose`-стиль: 13/19 sans, 14px между параграфами, max-width 720px. HTML строго sanitized (whitelist tags).
- **Цитата** (`> ...`) — `border-left: 2px solid var(--border-strong); padding-left: 12px; color: var(--fg-3);`.
- Attachments — grid 3-кол, 64×64 thumb или icon + filename + size.
- Actions внизу: `[Ответить] [Ответить всем] [Переслать]` ghost-buttons.

## Composer (новое письмо)

Это форма — см. также [04-forms-modals.md](04-forms-modals.md). Особенности:

- **From** — pre-selected box-аккаунт, fg-2 mono, не редактируется.
- **To/Cc/Bcc** — chip-input с автокомплитом (контакты + люди из карточки).
- **Subject** — обычный text, max 1 line.
- **Template selector** — dropdown сверху body: `Шаблон: КП клиенту ⌄`. Применение шаблона **подменяет** body после `confirm()` если уже есть текст.
- **Body** — rich text, минимум: bold / italic / link / list / quote / inline-attachment. Без цвета текста, без шрифтов, без размеров. Это деловая переписка.
- **Footer** — sticky с `[Сохранить черновик] [Запланировать ⌄]                     [Отправить]`.

## Sticky-правила (auto-reply / triage)

UI правил — см. готовый экран `ui_kits/crm/05-mail-rules.html`. Правило — это:

```
Если письмо подходит ┃ ОТ:    *@kone.com
                     ┃ ТЕМА:  содержит «refresh» «КП»
                     ┃ ВЛОЖ.: есть .pdf
                     ─
То                   ┃ привязать к заявке (по коду MZ-* в теме)
                     ┃ статус → «КП получено»
                     ┃ назначить → менеджер по бренду KONE
                     ┃ уведомить → менеджер
                     ─
Пометить sticky      ┃ ✓ да (бренд KONE → менеджер Иванов)
```

- Каждый блок — section card с рамкой.
- Условия — KV (`label: value`), не таблица.
- Тестировать правило кнопкой `Прогнать на последних 50 письмах`. Результат — список mail-row с пометкой `подошло / не подошло / частично`.

## Copywriting

- **Subject шаблона** — структурно: `[тип] [код] · [главный факт] · [срок/сумма]`. Пример: `КП MZ-21384 · 4 поз. · 142 800 ₽ · до 28.11`.
- **Preheader** — то, что человек видит в списке писем. Полезный факт, не дубль темы. `Действительно до 28.11. Срок 3 рабочих дня.`
- **Greeting** — `Здравствуйте, [имя]` или `Здравствуйте.` если имени нет. **Не** `Уважаемый/ая`, не `Привет`.
- **Sign-off** — короткая дефис-подпись на отдельной строке `—`, дальше имя · должность · контакты. Без `С уважением,`.
- **Кнопка** — глагол + объект: `Согласовать и оплатить`, `Открыть в CRM`. Не `Подробнее`, не `Перейти`.
- **Auto-reply** — пишется как от автомата, без эмодзи, **с подписью `автомат от MyLift CRM`**.

## Do / Don't

| ✅ Do | ❌ Don't |
|---|---|
| 600px table-layout, inline styles | flex/grid, `<style>` block |
| 1 CTA на письмо, accent | 3+ кнопки в письме |
| Тема структурно: `[тип] [код] · …` | Тема `Re: Re: Re: ваш запрос` |
| Sign-off на новой строке после `—` | `С уважением, команда MyLift!` |
| Sticky-уведомление спокойным тоном | `!!! СРОЧНО ОТВЕТЬТЕ КЛИЕНТУ !!!` |
| Цитата `>` border-left fg-3 | Цитата на цветном фоне |
| Шрифт system-ui stack | Inter / любой кастомный шрифт |

## Edge cases

- **Письмо без темы** — `(без темы)` fg-3 italic.
- **Очень длинный thread** (>20 писем) — collapse «Показать ещё 14 писем» в середине треда.
- **Внешние изображения** — блокировать по умолчанию, кнопка `Показать изображения` сверху body.
- **Tracking pixel** — не вставлять. CRM фиксирует open через свой mechanism.
- **Подпись клиента** в reply — отделять `--`-line и складывать в свёрнутый блок «Подпись».
- **Discard черновика** — `confirm()` если в body есть > 20 символов.
- **Шаблон с переменной {{client_name}}** — рендерим серым `{{client_name}}` placeholder, если значение не подставилось; не падаем.
- **Отправка не удалась** — письмо остаётся в `Исходящие` со статусом `не отправлено · [повторить]` chip red.
