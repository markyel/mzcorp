# 01 · List view (table view)

> Списки заявок, клиентов, поставщиков, писем, артикулов, sticky-правил, журналов событий.

## Когда применять

Пользователь говорит:
- «список заявок», «все заявки», «таблица клиентов»
- «очередь на распределении», «лента входящих»
- «инбокс поставщика», «список открытых КП»

Если в карточке внутри карточки нужен мини-список (позиции в заявке, история операций) — **это не list view**, это компонент `data-table` внутри detail. См. [02-detail.md](02-detail.md).

## Слои сверху вниз

### 1 · Topbar (общий)
Не редактируй на странице списка. См. готовый блок в `ui_kits/crm/01-pool.html`. Только подсвечивай нужный пункт rail.

### 2 · Context list (240px sidebar)
- **Группы** с `group-label` 10.5px uppercase fg-3.
- **Внутри группы** — пункты `.q` с иконкой 14px (моно-символ или Lucide), лейбл, счётчик.
- **Активный пункт**: `inset 2px 0 0 var(--accent)` слева, фон `--bg-surface` (приподнятый над `--bg-sidebar`).
- **Счётчик** обычный — моно 11.5px fg-3. Если требует внимания (просрочено, нераспределённые) — `pill` (`background: var(--red-50); color: var(--red-700)`).
- **Saved views** — последняя группа, последний пункт всегда `+ Сохранить текущий вид…` пунктиром fg-3.

**Почему:** левая колонка — это primary navigation для одной сущности. Группы должны читаться как «Мои → Команда → Сохранённое». Если групп нет, пользователь теряется среди 20+ очередей.

### 3 · Page header
- Crumbs — uppercase 11.5px fg-3, разделитель `·`. Никогда `>` или `/`.
- H1 — `Заявки в работе`, после пробела `·` и количество в fg-3 14px medium. Например: `Все открытые заявки · 147`. Это даёт сразу размер выборки, не заставляя считать.
- Meta-строка — `обновлено HH:MM · автообновление 60 с · 6 менеджеров онлайн · ₽ 4,12 М в работе`. Разделитель `·` (middot). Только факты, не ободрение.
- Справа: вторичный CTA (Экспорт CSV) → seg-control вида (Таблица / Канбан / Календарь).

### 4 · Filter bar
- Высота 44px (10px y-padding). Фон — `--bg-surface-2`.
- **Чип в состоянии on**: `background: var(--sky-50); border: 1px solid var(--sky-500); color: var(--sky-700);` + `×` справа.
- **Чип в состоянии off**: `border: 1px solid var(--border-strong); background: var(--bg-surface);` без `×`.
- **Add-фильтр**: `border-style: dashed; color: var(--fg-3);` с подписью `+ фильтр`.
- Справа — sort/group selector в виде `Сортировка: возраст ↓` (label fg-3, value fg-1 medium).

**Don't:** не делай dropdown с галочками для чипов. Каждый чип — конкретное значение фильтра.

### 5 · Bulk action bar
Появляется только когда `selectedCount > 0`. Иначе вообще не рендерится.

- Фон `--sky-50`, 1px `--border` снизу.
- `<info>` — `3 заявки выделено` (font-weight 600, 12.5px).
- `<meta>` — `· общая сумма … · средний возраст …` opacity 0.7.
- Действия — `.btn` с `border-color: var(--sky-500); color: var(--sky-700)` (вторичный sky-стиль).
- Опасное (закрыть/удалить) — отдельный `.btn.danger` (red-300 border, accent text), всегда последним справа.
- В самом конце — `Снять выделение` подчёркнутая пунктиром.

**Edge case:** если выделено > 50, добавляй `+ ещё фильтр…` — некоторые операции (массовый refresh) ограничены. Это правило — слой бизнес-логики, см. foundation doc.

### 6 · Group header (опционально)
Когда сортировка задаёт группы (по статусу, по менеджеру, по дате) — рендери sticky-заголовок:

```html
<div class="gh">
  <span class="caret">▼</span>
  <h3>Просрочено по SLA</h3>
  <span class="n">· 5</span>
  <span class="summary"><span>cр. возраст 4,2 дн</span><span>сумма 312 800 ₽</span></span>
</div>
```

- Высота строго 32px, padding 10px 20px 8px.
- Заголовок — uppercase 12px semibold.
- Сводка справа — fg-3 11.5px medium, разделители — пробелы.

### 7 · Table

#### Колонки (типичный набор)

| Колонка | Ширина | Тип |
|---|---|---|
| Checkbox | 24px | фикс |
| Код (ID) | 110px | моно 12px fg-2 |
| Название/тема | `minmax(220px, 1fr)` | главная, t1 + t2 (12.5/11.5) |
| Статус | 150px | chip |
| Менеджер | 130px | avatar 22px + имя |
| Клиент | 160px | t1 + t2 (имя + email/inn) |
| Сумма | 100px | num right · tnum |
| Возраст | 80px | mono 12px fg-3 (warn → amber, over → red) |
| Меню | 32px | `···` row menu trigger |

Для других сущностей (клиенты, поставщики) меняй средние колонки, но **первые две (CB + ID-моно) и две последние (числовая + меню) — обязательный шаблон**.

#### Состояния строки
- **Default**: 36px, `--border-subtle` снизу.
- **Hover**: `--bg-hover`. Курсор pointer.
- **Selected**: `--bg-selected` + `inset 2px 0 0 var(--sky-500)`.
- **Compact** (для длинных списков ≥ 100 строк по умолчанию): высота 28px, `t1` без `t2` (тогда колонка `title` — одна строка).

**Don't:** не делай checkbox круглым. Не используй красный для selected — он зарезервирован под бренд-CTA.

### 8 · Footer
32px, фон `--bg-surface-2`, моноширинный счётчик и pager справа.
- Активная страница pager — фон `--accent`, белый текст.
- Селектор `20 / стр.` справа.

## Плотность — варианты

| Вариант | Когда | Row | Padding cell | Title |
|---|---|---|---|---|
| **comfortable** | первый список (онбординг), мобильный | 44px | 12px x · 10px y | t1 + t2 + аватар-метка |
| **default** | продакшен | 36px | 12px x · 8px y | t1 + t2 |
| **compact** | списки ≥ 100, монитор ≥ 1440 | 28px | 12px x · 4px y | только t1 |

Переключатель в `pageh`-seg или (лучше) пункт меню «Плотность» в шестерёнке справа.

## Copywriting

- **H1** — императив-существительное, не глагол: `Заявки в работе` (не `Управляйте заявками`).
- **Empty state**: одно предложение, факт + действие, без эмодзи.
  - `Все заявки разобраны. Хорошая работа.`
  - `Нет нераспределённых заявок. Следующий цикл — через 22 мин.`
- **Loading**: `Загружаем 147 заявок…` (с числом, если оно известно).
- **Error**: `Не удалось получить список. [Повторить]`. Inline, не модалка.
- **Bulk-confirm**: всегда с числом и существительным в нужной форме: `Закрыть 7 заявок как «не наша тема»?`
- **Тоасты**: `Назначено: Иванов А.` / `7 заявок переподчинены Петрову С.` — без восклицания.

## Do / Don't

| ✅ Do | ❌ Don't |
|---|---|
| Один accent (red) — только на active CTA | Подсвечивать selected красным |
| Status — внутри chip, lowercase | Статус заглавными как Badge |
| Числа в правой колонке, tnum | Левое выравнивание сумм |
| Hover row → `bg-hover` | Hover row → border-color change |
| Sticky thead + sticky group header | Прокручиваемый header |
| Empty state — sentence + dot | Empty state с illustration / emoji |
| `···` row menu в фиксированной 32px колонке | Кнопки действий в каждой строке |

## Edge cases

- **Длинное название** в `t1`: `white-space:nowrap; overflow:hidden; text-overflow:ellipsis;` Полный текст — в tooltip.
- **Нет клиента** (orphan): в колонке клиента — `— не привязан` fg-3 italic; clickable → открыть quick-attach.
- **Нет суммы** (КП ещё не сформировано): `КП не сформ.` fg-3 muted, не `0 ₽`.
- **Нераспределённое**: avatar — пунктирный круг с `?`, имя — `не назначено` fg-3.
- **Просрочено > 7 дн**: возраст красным **+** chip-статус red, не два разных красных оттенка.
- **Sticky-цепочка**: badge `sticky` violet после названия, не отдельная колонка.
- **Вложения**: badge `📎 N` в title-строке (используй символ скрепки, не emoji-иконку).
- **>100 строк отфильтровано**: подсказка под filter bar `Показано 100 из 412. [Загрузить все]`.
