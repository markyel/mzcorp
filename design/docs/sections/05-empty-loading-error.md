# 05 · Empty / Loading / Error states

> Состояния «нет данных», «грузим», «упало». Применяй ко всему — спискам, карточкам, секциям, виджетам.

## Когда применять

- «пустое состояние», «empty state», «когда ничего нет»
- «loading», «скелетон», «прелоадер»
- «ошибка», «error», «не загрузилось»

Это не отдельные экраны, а **состояния** уже существующих экранов. См. также:
- список — [01-list.md](01-list.md), секция Copywriting
- карточка — [02-detail.md](02-detail.md), edge cases
- дашборд — [03-dashboard.md](03-dashboard.md), edge cases

## Принцип

Все три состояния отвечают на одну и ту же вилку у пользователя: **что произошло и что мне делать**. Разница только в причине:

| Состояние | Причина | Тон |
|---|---|---|
| Empty | данных нет (ещё или всё закрыто) | нейтрально-фактически |
| Loading | данные грузятся | без хайпа, не «магия» |
| Error | данные не пришли | уверенно, без паники |

Никогда не используй эмодзи и иллюстрации в data UI. Это CRM, не consumer-app.

## Empty state

### Правила

- **1 строка факта** + **1 строка действия (или CTA)**.
- Без картинок, без иконок-героев. Максимум — точечный 14px outline-icon перед заголовком, и тот часто лишний.
- Текст fg-2 13px regular, action — link-стиль `--sky-700` underline или primary 28px button.
- Зона занимает 33–50% высоты родителя, выровнена по центру.
- Padding минимум 24px со всех сторон.

### Шаблоны (выучить)

```
Empty list (всё разобрано):
  ┌───────────────────────────────────┐
  │                                   │
  │   Все заявки разобраны.           │
  │   Хорошая работа.                 │
  │                                   │
  └───────────────────────────────────┘

Empty list (никогда не было):
  ┌───────────────────────────────────┐
  │                                   │
  │   Заявок пока нет.                │
  │   [Создать первую]                │
  │                                   │
  └───────────────────────────────────┘

Empty filter (есть данные, но фильтры всё скрыли):
  ┌───────────────────────────────────┐
  │   Под текущие фильтры ничего      │
  │   не подходит.                    │
  │   [Сбросить фильтры]              │
  └───────────────────────────────────┘

Empty search:
  ┌───────────────────────────────────┐
  │   Ничего не нашли по «D320 KONE». │
  │   [Искать в архиве]               │
  └───────────────────────────────────┘

Empty section внутри карточки (одна строка):
  ─ Позиций ещё нет. [Добавить из каталога]

Empty timeline:
  ─ Событий пока нет.

Empty chart:
  Нет данных за выбранный период.
```

### Не делай

- ❌ `Здесь будут отображаться ваши заявки 🚀` — иллюстрационная подсказка как фича
- ❌ `Упс! Ничего не найдено` — извинение
- ❌ Иллюстрация коробки / лупы / inbox-zero
- ❌ 3 параграфа объяснения, что такое «заявка»

## Loading state

### Уровни

| Что грузим | Решение |
|---|---|
| Страница целиком (initial) | skeleton всей сетки (header + table + sidebar) |
| Список / таблица | skeleton-rows × 6 |
| Карточка / секция | skeleton-секции (44px header bar + 3 серых полоски) |
| Inline в кнопке | spinner 12px внутри кнопки, текст `[Создаём…]` |
| Refresh уже видимого списка | top-progress-bar 2px sky, не replace skeleton |

### Skeleton — правила

- Цвет — `--bg-surface-2` (тот же, что у hover). Не серый-чёрный, не градиент.
- Закругления и размеры — те же, что у реального компонента (квадрат → квадрат, текст → 8px высота закруглённый).
- **Shimmer-анимация**: левый-к-правому linear-gradient 1.4s, opacity 0.5 → 1 → 0.5. Не pulse fade-in/out (он визуально ломкий).
- Не больше 6 skeleton-rows. После 1.5с — оставить, не наращивать.
- Skeleton **не клонирует** контент. Это абстрактные плашки, не «фейковая строка с фейковым именем».

```css
.skel{ background: var(--bg-surface-2); border-radius: 4px; height: 8px; position: relative; overflow: hidden; }
.skel::after{ content:""; position:absolute; inset:0;
  background: linear-gradient(90deg, transparent, rgba(255,255,255,.6), transparent);
  animation: shimmer 1.4s linear infinite;
}
@keyframes shimmer{ 0%{transform:translateX(-100%);} 100%{transform:translateX(100%);} }
```

### Spinner — правила

- Только в inline-кнопках и в очень узких местах (<60×60px зона).
- 12px / 14px размер, border 1.5px, accent (`--accent`) для primary, fg-3 для secondary.
- Никогда не центрируй spinner на пустой странице. Это бессмысленно — пользователь не видит, что грузится.

### Текст под прогрессом

Если операция > 1с — текстовый прогресс:
```
Загружаем 147 заявок…
Шаг 2 из 4: проверяем поставщиков
Импортируем 12 380 строк прайса · 4 200 / 12 380
```

С числом, если оно известно. Без `Подождите, пожалуйста`.

## Error state

### Уровни

| Где упало | Решение |
|---|---|
| Один запрос (этот раздел) | inline-баннер на месте контента |
| Действие (submit формы) | inline-баннер сверху формы |
| Сетка целиком (network down) | top-баннер всей страницы (sticky, амбер) |
| Запрещено (403) | inline-баннер с конкретной причиной |
| Не существует (404) | страница не упала; рендерим её, но в карточке `Объект не найден` |

### Inline-баннер

```
┌─────────────────────────────────────────────────────────┐
│ ⚠ Не удалось получить список.        [Повторить]   [×] │
│   Попытка 2 из 3 · следующая через 18 с                 │
└─────────────────────────────────────────────────────────┘
```

- Фон `--red-50`, border 1px `--red-300`, padding 12px 16px.
- Иконка слева 14px outline (треугольник или круг). НЕ красная заливка-emoji.
- Текст fg-1 13px medium первой строкой, fg-3 11.5px второй (детали).
- Action `[Повторить]` — secondary button с `border-color: var(--accent); color: var(--accent)`.
- `[×]` справа — отбросить, баннер скрывается, контент остаётся пустой.

### Top-баннер (network)

Когда оффлайн или CRM-API лежит:

```
─────────────────────────────────────────────────────────
 ⚠ Соединение потеряно. Работаем по последним данным.
   Изменения будут отправлены, когда связь вернётся.
─────────────────────────────────────────────────────────
```

- Position sticky top, width 100%, height 36px, padding 8px 20px.
- Фон `--amber-50`, нижний border `--amber-300`. Это амбер, не red — значит «деградировало», а не «сломалось».
- Текст fg-2 12.5px medium, без CTA (повтор автоматический).

### Не делай

- ❌ `Что-то пошло не так. Попробуйте позже.` — это пустая фраза
- ❌ Отображать stack trace или JSON-error в production
- ❌ Полноэкранная заглушка `Ошибка 500` с гифкой
- ❌ Молча показывать пустое состояние — пользователь подумает, что данных нет

### Шаблоны сообщений

| Ситуация | Сообщение |
|---|---|
| Network timeout | `Не удалось получить список. Попытка 2 из 3.` |
| 403 forbidden | `Нет доступа к этому разделу. Запросить у админа?` |
| 404 not found | `Заявка MZ-99999 не найдена. Возможно, удалена.` |
| Validation | конкретное поле, см. [04-forms-modals.md](04-forms-modals.md) |
| Conflict | `Запись изменена другим пользователем. [Загрузить актуальную]` |
| Server 500 | `Сервер вернул ошибку. Мы уже знаем, попробуйте через минуту.` |

## Do / Don't

| ✅ Do | ❌ Don't |
|---|---|
| Empty: 1 строка факта + действие | Empty: иллюстрация + 3 параграфа |
| Loading: skeleton сетки | Loading: spinner на пустой странице |
| Error: inline-баннер на месте контента | Error: модалка поверх всего |
| Текст «Загружаем 147 заявок…» с числом | Текст «Подождите, пожалуйста…» |
| Network down → амбер top-баннер | Network down → red full-screen |
| Skeleton ровно 6 rows max | Skeleton наращивается до 50 rows |
| Retry — secondary с accent border | Retry — primary red |

## Edge cases

- **Optimistic update упал** — строка остаётся, но получает `border-left: 2px solid var(--amber-500)` + chip `не сохранено [повторить]`. Не убирай строку.
- **Разрешения сменились** — на следующий запрос показываем 403-баннер с действием `Перезайти`.
- **Slow connection** — после 5с скелетона добавляем text `Соединение медленное…` fg-3 под скелетоном.
- **Парциальный успех bulk-операции** — `Обновлено 45 из 50. 5 не удалось. [Посмотреть]` — амбер inline-баннер, не red.
- **Refresh во время скролла** — top-progress-bar 2px sky, не replace списка скелетоном (это вызывает раздражающий jump).
- **Авто-retry** — счётчик `следующая попытка через 18 с` в баннере, отсчёт mono.

## Checklist

- [ ] Empty — 1 строка факта + 1 действие, без иллюстрации
- [ ] Loading — skeleton, не spinner (кроме inline-кнопки)
- [ ] Error inline — на месте контента, не модалка
- [ ] Network down — амбер top-sticky, не red
- [ ] Текст прогресса с числом, если оно известно
- [ ] Conflict — inline-link `Загрузить актуальную`
- [ ] Skeleton не клонирует контент (абстрактные плашки)
- [ ] Никаких `Упс!`, `Что-то пошло не так`, `Подождите`
