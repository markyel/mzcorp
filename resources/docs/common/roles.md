# Роли и зоны ответственности

В системе 4 бизнес-роли + 1 техническая.

| Роль | Что делает | Что НЕ может |
|---|---|---|
| **Менеджер** (`manager`) | Ведёт свой пул заявок: общается с клиентами, готовит КП, контролирует оплату. Видит только свои. | Переподчинять, менять правила почты, видеть чужие заявки. |
| **РОП** (`head_of_sales`) | Управляет распределением: переподчиняет, разбирает спорные кейсы (mail-review), настраивает правила почты, мониторит SLA. Ведёт свой пул заявок наравне с менеджером. | Управлять админ-учётками. |
| **Секретарь** (`secretary`) | Контролирует, что входящая почта попала в нужные руки: читает раздел «Почта» и mail-review, перенаправляет ошибочно отклонённые. Без операционной работы с заявками. | Менять статусы заявок, переподчинять, удалять. |
| **Директорат** (`director`) | Аналитика, стратегические решения, кураторство базы знаний. Просмотр всего. | Менять админ-учётки, активировать общие ящики. |
| **Админ** (`admin`) | Технический администратор: подключение общих ящиков, OAuth-привязки, разруливание сбоев. Видит всё, имеет все права. | — |

## Кто получает новые заявки

`AssignmentService` распределяет на менеджеров и РОПа (`requestHandlerRoles`). Секретарь / директор / админ — **не получают** заявок: они нужны для контроля и не работают с пулом операционно.

Если у менеджера/РОПа стоит ненулевая [плановая нагрузка](/docs/rop/managers), система учитывает её при выборе следующего адресата.

## Доступ к разделам

| Раздел | Менеджер | РОП | Секретарь | Директор | Админ |
|---|:---:|:---:|:---:|:---:|:---:|
| Дашборд | ✓ свой | ✓ полный | ✓ полный | ✓ полный | ✓ |
| Заявки (свой пул) | ✓ | ✓ | — | — | — |
| Заявки (все) | — | ✓ | ✓ | ✓ | ✓ |
| Поиск по каталогу | ✓ | ✓ | ✓ | ✓ | ✓ |
| Счета | ✓ свои | ✓ все | ✓ все | ✓ все | ✓ |
| Почта (read-only) | — | ✓ | ✓ | ✓ | ✓ |
| Правила почты | — | ✓ | — | ✓ | ✓ |
| Авто-отклонённые | — | ✓ | ✓ | ✓ | ✓ |
| Менеджеры | — | ✓ | — | ✓ | ✓ |
| Настройки | — | ✓ | — | ✓ | ✓ |
| Общие ящики | — | — | — | — | ✓ |

Прочерк не означает «запрещено для жизни» — означает, что страница не рендерится или возвращает 403.
