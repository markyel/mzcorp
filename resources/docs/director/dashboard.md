# Дашборд — как читать

`/dashboard` — сводный экран. Для директора и РОПа — полная версия с командными метриками. Для менеджера — урезанная (только свои показатели + общий health).

Период переключается в шапке: **7 / 30 / 90 дней**.

## Верхний ряд KPI

| Метрика | Что считает |
|---|---|
| **Заявок в работе** | Все open-статусы (не paused/closed/paid/pending). |
| **Time-to-quote (среднее)** | Сколько часов в среднем проходит от создания Request до отправки первого КП. |
| **Просрочено по SLA** | Заявки с `attention_level=1`, исключая `client_replied` (он amber, не алярм). |
| **Конверсия КП → счёт** | % заявок, дошедших от `quoted` до `invoiced` за период. |
| **Сумма открытых КП** | Сумма `total_amount` всех `outbound_quote` со статусом matched, у которых заявка ещё не закрыта. |

Каждый KPI имеет дельту к предыдущему периоду (+12 за неделю, -1.2 пп и т.п.) и иногда цель (cель ≤ 4 ч, цель 50%).

## Sparklines (трендовые линии)

Маленькие графики за выбранный период:

- **Входящие письма** — поток клиентских запросов.
- **Авто-обнаружено КП** — сколько раз DocumentDetector успешно распознал исходящее.
- **Распределено sticky** — какая доля заявок ушла по sticky (а не round-robin).
- **Refresh по поставщикам** — сколько запросов цены ушло, какой % получил ответ.

## Загрузка менеджеров

Бар-чарт по каждому менеджеру:
- Сегменты — открытые заявки по статусам (в работе / ждёт клиента / на паузе / просрочено).
- Цифра справа — total open.
- Процент — `effective_load / pool_avg` (где у менеджера загрузка относительно среднего по команде). 100% — норма, выше — перегружен, ниже — недогружен.
- «Пик · NN%» — пиковое значение за период.
- Менеджеры в отпуске показаны отдельно с пометкой «отпуск».

Полезно для решений о найме / переподчинении.

## Воронка

Сколько заявок дошло до каждого статуса за период:

```
Получено N писем → N заявок → N квалифицированных → N КП → N счетов → N оплачено
```

Стрелки между этапами с %-конверсией. Помогает увидеть, где «течёт».

## Поток входящих по часам · будни

Heatmap N × 24 (5 рабочих дней × 24 часа). Цвет ячейки = количество писем в этот час. Помогает спланировать рабочие смены — когда пик нагрузки.

## Лента распределения

Live-лента последних действий диспетчера (auto + ручные):

```
14:38:12  MZ-21384 назначено Иванов А. по sticky-правилу catalog_item_id=M27878
14:35:47  Refresh цен запрошен у KONE-RU и SEC-Slovenia. 6 позиций, batch #R-1142
14:33:21  ...
```

Полезно для проверки гипотез «почему именно этому менеджеру». Каждая запись — это `RequestAssignment::reason` поле (см. [Распределение](/docs/rop/distribution)).
