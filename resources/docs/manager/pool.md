# Мой пул заявок

Раздел **Заявки** — это таблица с группировкой по статусам. Слева — узкая навигация с сохранёнными запросами и счётчиками, сверху — фильтры и поиск. Клик по строке открывает карточку заявки на отдельной странице.

## Что в пуле

В пул попадают все заявки, у которых вы — текущий ответственный (включая временные делегации, если коллега в отпуске передал свою заявку на вас).

Заявки приходят сюда тремя путями:

1. **Автоматическое распределение.** Письмо клиента приходит на общий ящик (`mail@myzip.ru`, `info@myzip.ru`), система парсит позиции и распределяет — sticky-правило (один клиент или артикул → один менеджер) + round-robin по нагрузке.
2. **Ручное переподчинение РОПом** через карточку → `Переподчинить`.
3. **Перевод с отсутствующего коллеги** — заявки уходящего в отпуск автоматически перетекают на дежурного.

Заявки в статусе **«В обработке»** (парсер позиций ещё работает) менеджеру не показываются — там нечего делать. РОП/секретарь/директор их видят для контроля.

## Левая навигация (240 px)

| Группа | Пункт | Что отбирает |
|---|---|---|
| **Мои · {имя}** | **Мои в работе** | Все ваши открытые заявки (без paused / closed). Счётчик справа. |
| **Команда** *(только для РОПа / директора / секретаря / админа)* | **Нераспределённые** | `assigned_user_id IS NULL` — заявки, которым ещё не назначен менеджер. |
| | **Все открытые** | Все открытые заявки всех менеджеров. |

Ниже — серые заглушки Phase 2: «Жду клиента», «На паузе», «КП отправлено», «Счёт выставлен». Пока некликабельные, эквивалент достигается комбинацией bucket + status.

## Bucket-фильтры (сверху таблицы)

Группа статусов, к которой применяются дальнейшие фильтры.

| Bucket | Что входит |
|---|---|
| **Активные** | Все открытые статусы (от Новой до Оплачено). По умолчанию. |
| **Просрочено** | Только заявки с `attention_level=1` в открытых статусах, кроме «клиент ответил» (это не алярм). |
| **На паузе** | Только `paused`. |
| **Закрытые** | `closed_won` + `closed_lost`. |
| **Все** | Всё, что вам видно (менеджеру — без «В обработке»). |

Рядом со счётчиком каждого bucket'а можно дополнительно отфильтровать по конкретному статусу — chips с count'ами внутри выбранного bucket'а.

## Поиск

Поле `⌘K` сверху ищет одновременно по:

- коду заявки (`M-2026-1488`),
- теме письма,
- имени клиента и его e-mail,
- артикулу и названию позиции (включая распарсенные `parsed_article` / `parsed_name`),
- M-SKU каталога (через привязку `catalog_item_id`),
- коду исходящего КП (`КП-2026-NNNN`).

## Таблица

Колонки: чекбокс · код · заявка (тема + бейджи) · статус · менеджер · клиент · сумма · возраст · ⋯

Строки сгруппированы по статусу с sticky-заголовками. Порядок групп — по lifecycle (Новая → В работе → Жду клиента → КП → Согласован → Счёт → Оплачено → Закрыта).

**Сортировка:**

- Менеджер всегда видит **attention-первая** (просроченные сверху + последняя активность). Переключить нельзя.
- РОП / директор / секретарь / админ могут переключаться на «по дате создания» (свежие / старые) — для разбора backlog.

## Attention-маркер

Подсветка строки + сдвиг наверх. Поднимается системой автоматически, когда от вас ждут действия:

- **Просрочено** — превышен SLA на следующий шаг.
- **Клиент ответил** — пришла реплика по треду (amber-подсветка, но не алярм — поэтому не в bucket «Просрочено»).
- **Пауза закончилась** — `paused_until` истёк, заявка вернулась в работу.
- **Новая** — заявка только что назначена, ещё не открывали.

Attention снимается, как только вы открыли заявку, сменили статус или отправили ответ.

## Что нельзя

- Перетянуть чужую заявку себе. Это делает РОП.
- Удалить заявку. Только закрытие (успех / потеря / не наша тема).
