# Работа с заявкой

Заявка — это отдельная страница `/dashboard/requests/{id}`, открывается кликом по строке в пуле. Состояние меняется кнопками в карточке — это синхронизирует attention-маркер, SLA-таймеры и отчётность.

## Карточка: что где

**Сверху — hero-блок**: тема, статус-чип, код заявки (`M-2026-1488`), клиент, основные метрики (сумма по сматченным позициям, возраст, кто менеджер). Если парсер ещё работает — рядом с темой висит chip «парсится…», карточка сама обновится через 10 секунд.

**Ниже — табы.** Активные сейчас:

| Таб | Что показывает |
|---|---|
| **Обзор** | Сводный экран: превью позиций, превью переписки, сводка по клиенту, активность за последнее время. |
| **Переписка** | Полный тред: входящие/исходящие письма с вложениями, цитированные блоки сворачиваются. Здесь же — форма ответа. |
| **Позиции** | Распарсенные строки запроса (артикул, бренд, кол-во), привязка к каталогу, итог по КП и НДС. |
| **КП** | Черновики и отправленные коммерческие предложения. Редактор КП открывается здесь. |
| **Счета** | Счета, выставленные по этой заявке. |
| **Активность** | Audit-журнал: назначения, переподчинения, смены статуса, действия системы (auto-detect КП, resume с паузы и т.п.). |
| **Файлы** | Все вложения по заявке в одном списке. |

Отдельно в наборе видны **Поставщики** и **Связанные** — они помечены как `disabled` (Phase 2).

## Статусы

Все 14 статусов и их смысл. Жирным — те, что используются ежедневно.

| Статус | Что означает |
|---|---|
| В обработке | Парсер ещё разбирает письмо. Менеджеру не показывается в пуле. |
| **Новая** | Назначена, ещё не открывали. |
| Назначена | Открывали, но к работе не приступили. В пуле объединена с «В работе» в одну группу (для упрощения чтения). |
| **В работе** | Активно ведёте: уточняете позиции, ищете в каталоге, готовите КП. |
| **Жду клиента** | Отправили вопрос — ждём ответ. |
| **КП отправлено** | Коммерческое ушло, клиент думает. |
| На согласовании | Клиент сам отметил, что «обсуждаем внутри». |
| Отложена | Клиент попросил вернуться позже (конкретная дата). |
| **Согласован / ждёт счёт** | Клиент согласился — нужен счёт. |
| **Счёт отправлен** | Счёт ушёл. Ждём оплату. |
| Оплачено | Деньги пришли. Закрываем как успех. |
| **На паузе** | Заморожена до конкретной даты (мета-статус). |
| **Закрыто · успех** | Сделка завершилась продажей. |
| **Закрыто · потеря** | Сделка не состоялась (с причиной). |

## Переходы

Кнопки в правой панели меняются под текущий статус. Стандартный путь:

```
Новая → В работе → (Жду клиента ↔ В работе) →
КП отправлено → Согласован / ждёт счёт →
Счёт отправлен → Оплачено → Закрыто · успех
```

В любом активном статусе можно прыгнуть напрямую на:
- **КП отправлено** — если детектор не распознал КП в исходящем (обсудили устно, КП в WhatsApp).
- **Счёт отправлен** — если счёт ушёл повторному клиенту без формальной цепочки КП → согласование.
- **Закрыто · успех** — для cash без формального учёта оплаты.
- **Закрыто · потеря** — отказ или «не наша тема». Нужно выбрать причину из списка.

## Пауза

Если ясно, что заявка зависнет на день-три (поставщик в отпуске, ждём фото от инженера), не таскайте её в attention-листе. Жмите **Пауза**, выберите дату возобновления и причину.

- Максимальная пауза — **21 день** (настраивается РОПом).
- В день `paused_until` заявка автоматически возвращается в прежний статус, attention поднимается.
- Пауза видна в hero: «На паузе до 15.01 · поставщик в отпуске».

## Переподчинить

Кнопка `Переподчинить` доступна РОПу и админу. Менеджер сам себе заявку не переподчиняет.

Если уходите в отпуск — заполните период в профиле (`unavailable_from / unavailable_until`). Активные заявки автоматически передадутся дежурному; в день возвращения вернутся обратно.

## Что делает система автоматически

- **Распределяет** новые заявки по правилам (sticky → round-robin по нагрузке).
- **Парсит** позиции из письма и вложений (артикул, бренд, кол-во), подтягивает каталог.
- **Детектирует исходящие КП и счета** — если вы отправили PDF в треде, статус заявки сдвигается сам.
- **Мониторит ответы клиента** — поднимает attention, как только пришла реплика.
- **Считает SLA** в рабочих часах (Пн-Пт 9-18 Москва) с учётом производственного календаря.
- **Возобновляет с паузы** в нужную дату.

Если что-то из этого выглядит не так (статус сдвинулся ошибочно, заявка не парсится) — сообщите через ▲ в шапке.

## Что закрыто

- Чужая заявка для редактирования. Только просмотр (если у вас есть права видеть «все»).
- Удаление позиций после отправки КП — только пометка «снято». История остаётся.
- Прямое редактирование каталога — каталог только из 1С.
