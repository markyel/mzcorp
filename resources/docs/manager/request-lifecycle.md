# Работа с заявкой

Каждая заявка проходит конечный набор статусов. Меняйте статус кнопками в карточке — это синхронизирует attention-маркер, SLA-таймеры и отчётность.

## Карточка: что где

Заявка открывается в правой панели пула или по прямой ссылке `/dashboard/requests/{id}`. Сверху — hero-блок с темой, клиентом, текущим статусом и кодом заявки (`M-2026-1488`). Ниже — табы.

| Таб | Содержимое |
|---|---|
| **Тред** | Все письма по заявке: вход/исход, attachments, parsed-результаты AI. |
| **Позиции** | Распарсенные строки запроса: артикул, бренд, количество. Связка с каталогом. |
| **КП** | Черновики и отправленные коммерческие предложения. |
| **Счета** | Счета, которые вы выставили (фаза 4+). |
| **Активность** | Audit-лог: назначения, переподчинения, смены статуса, действия системы. |

Справа в карточке — панель действий: смена статуса, пауза, переподчинение, отправить ответ.

## Статусы

Все статусы заявки и их смысл. Жирным — те, что используются ежедневно.

| Статус | Что означает |
|---|---|
| В обработке | Парсер ещё разбирает письмо. Менеджеру не показывается. |
| **Новая** | Заявка назначена, ещё не открывали. |
| **Назначена** | Открывали, но ещё не начали работать. |
| **В работе** | Активно ведёте: уточняете позиции, ищете в каталоге, готовите КП. |
| **Жду клиента** | Отправили ему вопрос — ждём ответ. |
| **КП отправлено** | Коммерческое предложение ушло, клиент думает. |
| На согласовании | Клиент сам отметил, что «обсуждаем внутри». |
| Отложена | Клиент попросил вернуться позже (конкретная дата). |
| **Согласован / ждёт счёт** | Клиент согласился — нужен счёт. |
| **Счёт отправлен** | Счёт ушёл. Ждём оплату. |
| Оплачено | Деньги пришли. Закрываем как успех. |
| **На паузе** | Заморожена до конкретной даты (мета-статус). |
| **Закрыто · успех** | Сделка завершилась продажей. |
| **Закрыто · потеря** | Сделка не состоялась (с причиной). |

## Переходы

Кнопки в правой панели меняются под текущий статус. Доступные переходы:

| Из | Куда | Когда жать |
|---|---|---|
| Новая → В работе | начали разбираться. |
| В работе → Жду клиента | задали вопрос — система пометит attention, когда придёт ответ. |
| В работе → КП отправлено | ушло коммерческое. Статус ставит либо вы вручную, либо детектор (распознаёт PDF/таблицу в исходящем письме). |
| КП → Согласован / ждёт счёт | клиент сказал «выставляйте счёт». |
| Согласован → Счёт отправлен | счёт ушёл. |
| Счёт → Оплачено → Закрыто · успех | деньги получены, сделка закрыта. |
| Любой ненулевой → Закрыто · потеря | клиент отказался, конкурент выиграл, заявка не наша тема. Нужно выбрать причину из списка. |

> **Escape hatch:** из любого активного статуса можно прыгнуть сразу на `КП отправлено`, `Счёт отправлен` или `Закрыто · успех`. Это нужно, если КП обсудили устно или счёт ушёл повторному клиенту без формальной цепочки.

## Пауза

Если ясно, что заявка зависнет на день-три (поставщик в отпуске, ждём фото от инженера), не таскайте её в attention-листе. Жмите **Пауза**, выберите дату возобновления и причину.

- Максимальная пауза — **21 день** (настраивается РОПом).
- В день `paused_until` заявка автоматически возвращается в прежний статус, attention поднимается.
- Пауза видна в hero-блоке: «На паузе до 15.01 · поставщик в отпуске».

## Переподчинить

Кнопка `Переподчинить` доступна РОПу и админу. Менеджер сам себе заявку не переподчиняет.

Если уходите в отпуск — заполните период в профиле (`unavailable_from / unavailable_until`). Активные заявки автоматически передадутся дежурному менеджеру; в день вашего возвращения они вернутся обратно.

## Что делает система автоматически

- **Распределяет** новые заявки по правилам (sticky → round-robin).
- **Парсит** позиции из письма (артикул, бренд, кол-во) и подтягивает каталог.
- **Детектирует исходящие КП и счета** — если вы отправили PDF в треде, статус заявки сдвигается сам.
- **Мониторит ответы клиента** — поднимает attention, как только пришла реплика.
- **Считает SLA** в рабочих часах (Пн-Пт 9-18 Москва) с учётом производственного календаря.
- **Возобновляет с паузы** в нужную дату.

Если что-то из этого выглядит не так (статус сдвинулся ошибочно, заявка не парсится) — сообщите через `связь с создателем` в шапке.

## Что закрыто

- Чужая заявка для редактирования. Только просмотр через дашборд РОПа.
- Удаление позиций после отправки КП — только пометка «снято». История остаётся.
- Прямое редактирование каталога — каталог только из 1С.
