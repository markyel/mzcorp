# Управление менеджерами

`/dashboard/managers` — список всех пользователей системы (менеджеры, РОПы, секретари, директорат). Создание и редактирование — `/dashboard/managers/create` и `/dashboard/managers/{id}/edit`.

## Список — колонки

| Колонка | Что показывает |
|---|---|
| **Имя** | + аватар с инициалами. |
| **Email** | Логин в систему. |
| **Роль** | manager / head_of_sales / secretary / director. Admin доступен только админу. |
| **Личный ящик** | Состояние OAuth-привязки. См. ниже. |
| **Статус** | в архиве / недоступен до DD.MM (отпуск) / план отсутствия / норм. |

### Колонка «Личный ящик»

| Состояние | Цвет | Когда |
|---|---|---|
| `email@…` | зелёный | подключён, работает. |
| `отвязан` | амбер | существует, но `is_active=false`. |
| `ошибка sync` | красный | `last_error_at` свежее `last_synced_at`. Проверить токены / перепривязать. |
| `—` | серый | менеджер/РОП без подключённого ящика. Можно подключить через карточку. |
| `не для этой роли` | серый | director/secretary/admin — личный ящик не подключается (синк не работает). |
| `не должен синкаться` | **амбер** | ящик подключен директору/секретарю/админу. **Нужно отвязать** — синк закупочной переписки попадает в пул как клиентские заявки. |

## Карточка менеджера

`/dashboard/managers/{id}/edit` — три блока сверху вниз:

### 1. Editor (ФИО, email, роль, пароль)

Без сюрпризов. Email — логин, пароль обязателен на create / опционален на edit. Менеджер сам меняет в `/profile`.

### 2. Плановая нагрузка (только для manager / head_of_sales)

Поле «Плановая нагрузка %», диапазон 10–500, шаг 10, дефолт 100.

- 100 — норма (стандартная доля).
- 50 — в 2 раза меньше заявок.
- 200 — в 2 раза больше.

Эффект — постепенный. Текущая накопленная нагрузка не сбрасывается, просто новые заявки идут с приоритетом «отстающим». См. [Распределение → Round-robin](/docs/rop/distribution).

### 3. Личный ящик (только для manager / head_of_sales)

Подключение Yandex 360 через OAuth flow:

1. **Если ящика нет** — форма «email + название». После save создаётся запись Mailbox с `type=personal`, `is_active=true`.
2. **Если ящик есть, но токенов нет** — кнопка-ссылка на Yandex authorize URL (откроется новая вкладка) + поле «Код подтверждения» из callback'а + `Save code`.
3. **Если токены есть** — статус (срок expiry), кнопки «Переподключить» (обнулить credentials) и «Отвязать» (мягкое `is_active=false`, токены не трогаем).

Для роли НЕ из `[manager, head_of_sales]` блок заменяется на info-плашку «Личный ящик не подключается для этой роли» — синхронизация работает только для тех, кто ведёт клиентские заявки.

## Отпуск / отсутствие

В карточке менеджера задаются `unavailable_from / unavailable_until` (диапазон дат) и `unavailable_reason` (свободный текст для chip). Если стоит `unavailable_auto_delegate=true`, по наступлению `unavailable_from` запускается cron `users:apply-planned-unavailability`, который открывает активные заявки этого менеджера дежурному.

В пуле дежурного они появляются с пометкой «временно от @Имя». В день возвращения откатываются обратно.

## Архивация

Кнопка «В архив» в списке. Менеджер с `archived_at != NULL`:
- Не получает новых заявок (`User::scopeAvailable` его исключает).
- Не виден в фильтре «по менеджеру» в пуле.
- Не виден в reassign-диалоге.
- Открытые заявки **остаются на нём** (статусы не трогаются). РОП должен явно переподчинить.

Восстановление — кнопка «Восстановить» (видна только если включить фильтр «архив»).
