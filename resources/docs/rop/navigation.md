# Где что — навигация РОПа

Шапка и рейл устроены так же, как у менеджера (см. [Менеджер → Где что](/docs/manager/navigation)). Дополнительно у РОПа в шапке появляются 4 пункта меню и расширенный workspace-пилл.

## Верхняя шапка

| Элемент | Что делает |
|---|---|
| **Workspace-пилл** `mail@…` | Видите название основного общего ящика + счётчик активных, цвет точки = здоровье инфраструктуры (зелёный — всё ок, янтарный — где-то ошибка sync). Менеджеру не показывается. |
| **Дашборд / Заявки** | Базовые пункты. |
| **Правила почты** | Настройка автоматической маршрутизации входящих писем. См. [Правила почты](/docs/rop/mail-rules). |
| **Авто-отклонённые** | Письма, которые AI классифицировал как irrelevant — вы можете пересмотреть и реоткрыть как заявку. См. [Авто-отклонённые](/docs/rop/mail-review). |
| **Менеджеры** | Управление учётками: создание, OAuth-привязка личных ящиков, отпуска, плановая нагрузка, архив. См. [Менеджеры](/docs/rop/managers). |
| **Настройки** | Override параметров системы: матчинг каталога, дедлайны SLA, поведение AI-детектора. См. [Настройки](/docs/rop/settings). |
| **🔔 / ❓ / ▲ / аватар** | Как у менеджера. |

## Левый рейл — дополнительно

Кроме общих 4 иконок (Дашборд / Заявки / Поиск по каталогу / Счета) РОП видит:

| Иконка | Раздел |
|---|---|
| ✉ | **Почта** — read-only листинг всей переписки по всем ящикам, в одном месте. Полезно при разборе спорных кейсов (например, почему линкер прицепил reply не туда). |

## Что РОП может, чего не может менеджер

- **Видеть и фильтровать чужие заявки.** В пуле есть фильтр «по менеджеру».
- **Переподчинять.** Кнопка «Переподчинить» в карточке заявки.
- **Реанимировать closed-заявки.** Кнопка «↻ Реанимировать» на карточке `closed_lost`.
- **Перезапускать парсер позиций.** Кнопка «↻ Перезапустить парсер» когда `items=0` и парсер уже отработал.
- **Сортировать пул по дате создания** (свежие/старые) — менеджеру эта опция не показывается, он всегда видит attention-first.
